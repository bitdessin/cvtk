Web Application
###############


Models for object classification, detection, or instance segmentation
can also be deployed as a web application.
The **cvtk** package provides commands to easily generate example code for this purpose.


Object Classification
*********************


First, refer to the :doc:`object classification tutorial <cls>` to build and train a model.
Assume the source code for object classification
generated by **cvtk** (using ResNet18 by default)is saved as :file:`cls.py`,
and the trained model's weights are saved in :file:`./outputs/fruits.pth`.

Then, follow the steps below to create a web application for the object classification model.

The ``cvtk app`` command is used to generate the source code for the web application.
This command requires the following arguments:

- ``--project``: The name of the project. A directory with this name will be created to store the web application source code.
- ``--source``: The source code file (e.g., :file:`cls.py`) generated by **cvtk** for object classification.
- ``--label``: The label file used for training the classification model.
- ``--model``: The model name used for object classification. The default is ResNet18 (``resnet18``). Change it if a different architecture is used.
- ``--weights``: The trained weights file (e.g., :file:`./outputs/fruits.pth`) of the classification model.


.. code-block:: sh
    
    cvtk app \
        --project fruits_cls_app \
        --source cls.py \
        --label ./data/fruits/label.txt \
        --model resnet18 \
        --weights ./outputs/fruits.pth


If the command runs successfully, a directory named :file:`fruits_cls_app` will be created,
and source code files for the web application will be generated in it.
Next, execute the following command to start the web application server (:file:`fruits_cls_app/main.py`):


.. code-block:: sh

    python fruits_cls_app/main.py --host 0.0.0.0 --port 8080


You can now access the object classification model through a web browser at
`http://localhost:8080 <http://localhost:8080>`_.



Additionally,
you can also modify the object classification source code
to be independent of the **cvtk** package.  
In this case, specify the ``--vanilla`` option when executing the ``cvtk app`` command.  
Note that the object classification source code itself must also be generated with the ``--vanilla`` option.


.. code-block:: sh
    
    cvtk create --script cls.py --task cls --vanilla

    # train the model with cls.py

    cvtk app \
        --project fruits_cls_app \
        --source cls.py \
        --label ./data/fruits/label.txt \
        --model resnet18 \
        --weights ./outputs/fruits.pth \
        --vanilla




Object Detection
****************

First, refer to the :doc:`object detection tutorial <det>` to build and train a detection model.  
Assume the detection source code generated by **cvtk** is saved as :file:`det.py`,  
and the trained model's weights are saved in :file:`./outputs/strawberry.pth`.

Then, generate the web application source code with the following command.


.. code-block:: sh
    
    cvtk app \
        --project strawberry_detapp \
        --source det.py \
        --label ./data/strawberry/label.txt \
        --model faster-rcnn_r101_fpn_1x_coco \
        --weights ./outputs/strawberry.pth


This will create a directory named :file:`strawberry_detapp`,
containing the web application source code.
Next, you can start the web application server as follows.


.. code-block:: sh

    python strawberry_detapp/main.py --host 0.0.0.0 --port 8080


Now, you can access the object detection model through a web browser at
`http://localhost:8080 <http://localhost:8080>`_.


.. image:: ../_static/demoapp.det.jpg
    :width: 85%
    :align: center


As with classification, the detection model can also be deployed without depending on **cvtk**.
First, generate detection source code with the ``--vanilla`` option, train the model,
and then generate the web application with ``cvtk app --vanilla``.

.. code-block:: sh
    
    cvtk create --script det.py --task det --vanilla

    # train the model with det.py

    cvtk app \
        --project strawberry_detapp \
        --source det.py \
        --label ./data/strawberry/label.txt \
        --model faster-rcnn_r101_fpn_1x_coco \
        --weights ./outputs/strawberry.pth \
        --vanilla




Instance Segmentation
*********************


First, refer to the :doc:`instance segmentation tutorial <segm>` to build and train a segmentation model.
Assume the segmentation source code generated by **cvtk** is saved as :file:`segm.py`,  
and the trained model's weights are saved in :file:`./outputs/strawberry.pth`.

Next, generate the web application source code with the following command.

.. code-block:: sh
    
    cvtk app \
        --project strawberry_segmapp \
        --source segm.py \
        --label ./data/strawberry/label.txt \
        --model mask-rcnn_r101_fpn_1x_coco \
        --weights ./outputs/strawberry.pth


This will create a directory named :file:`strawberry_segmapp`,
containing the web application source code.
Then, start the web application server as follows.


.. code-block:: sh

    python strawberry_segmapp/main.py --host 0.0.0.0 --port 8080



You can now access the instance segmentation model through a web browser at
`http://localhost:8080 <http://localhost:8080>`_.


To deploy the segmentation model without relying on **cvtk**,
first generate the segmentation source code with ``--vanilla``,
train the model, and then generate the web application with ``cvtk app --vanilla``.


.. code-block:: sh
    
    cvtk create --script segm.py --task segm --vanilla

    # train the model with segm.py

    cvtk app \
        --project strawberry_segmapp \
        --source segm.py \
        --label ./data/strawberry/label.txt \
        --model mask-rcnn_r101_fpn_1x_coco \
        --weights ./outputs/strawberry.pth \
        --vanilla
